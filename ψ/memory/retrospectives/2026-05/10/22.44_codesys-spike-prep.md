## Lessons Learned

1. **Enumerate design decisions as a numbered list in review request** — Atlas review ทำ 5/5 ใน comment เดียวเพราะ decisions ถูก list 1-5 + reasoning ต่อข้อ. ถ้าจมใน prose, reviewer ต้องไล่หา → review ช้า + อาจมองข้ามบางข้อ. Pattern: design-decision section as table { #, decision, reasoning, alternative-rejected }.

2. **Surface platform/environment constraints in the first response, not the second** — "Forge อยู่ WSL2, CODESYS = Windows GUI" คือ contract-defining fact. ถ้าปล่อยไป 1-2 turn ก่อนพูด, ทั้ง pacing + pacing + role assignment จะตั้งบนสมมติฐานผิด. รูปแบบ "Platform reality" table (4-5 rows: layer / where / who-acts) ใช้ซ้ำได้สำหรับ routed tasks ทุกประเภท.

3. **Project memory file early, not late** — เขียน `spike_codesys_chiller.md` หลัง batch 1 (ไม่ใช่ตอนปิดวัน) ทำให้ session ถัดไป pickup ตรง. Pattern: open multi-session work → first action after scope-confirm = write memory stub with file layout + pacing protocol + current state.
